## 1.3 核心背景

本系统核心应用于矿山场景，需实现“矿井地图渲染+设备坐标手动绑定”功能，解决不同矿山矿井地图格式不统一、坐标系差异大、设备坐标绑定繁琐的问题，依托边缘计算轻量化特性，实现地图快速加载、设备精准定位，兼顾2D全局预览与3D局部精确定位的需求，降低矿方配合成本，提升系统通用性和易用性。

# 2. 系统核心需求

## 2.1 总体需求

开发一套边缘计算空间数据驱动系统，支持矿山矿井地图的加载与渲染，实现设备图标的手动拖拽/点击绑定，支持2D全局视图与3D局部视图的无缝切换，自动获取绑定点位的坐标（2D：X、Y；3D：X、Y、Z）并存储，适配不同矿山的差异化地图格式和坐标系，确保系统轻量化、可通用、易操作。

## 2.2 地图相关需求

### 2.2.1 地图格式支持

- 支持矿山提供的主流地图格式：DWG、DXF（优先推荐）、Shapefile（.shp）、MapGIS文件（.wl/.wt/.wp）；不支持无坐标的PDF、PNG/JPG图片（若矿方仅能提供，需提示无法直接用于坐标绑定）。

- 系统需内置格式转换功能，将矿方提供的DWG/DXF/Shapefile等格式，自动转换为可直接渲染的GeoJSON格式（2D），以及适配3D渲染的格式（GLB/3D Tiles，局部场景）。

- 地图加载需轻量化，适配边缘计算设备，2D地图（GeoJSON）加载时间≤3秒，3D局部视图加载时间≤5秒。

### 2.2.2 坐标系适配

- 支持矿山独立坐标系（一矿一套），无需用户手动配置坐标系参数，系统自动从转换后的地图文件中读取坐标范围（minX、maxX、minY、maxY、minZ、maxZ）。

- 确保2D视图与3D视图使用同一套矿井坐标系，实现坐标无缝映射（2D的X、Y与3D的X、Y完全一致，3D新增Z轴高程信息）。

- 兼容国家统一基准（CGCS2000国家大地坐标系、1985国家高程基准），矿方提供的地图若基于老基准（北京54、西安80），系统无需额外处理，仅需保证地图内部坐标统一即可。

### 2.2.3 视图渲染需求

- 2D视图：渲染矿井总图（巷道、硐室、设备布置区域等核心要素），支持地图缩放、平移，清晰显示矿井空间布局，标注关键区域（如中央变电所、工作面、井底车场）。

- 3D视图：仅在2D视图某点被激活后渲染，为该点对应的局部小场景（如某段巷道、某个硐室），支持3D旋转、缩放、平移，清晰展示空间层高、层位关系，便于设备3D坐标精准绑定。

- 视图切换：2D视图点击激活点后，自动弹出/加载对应的3D局部视图；3D视图可返回2D总图，切换过程中保持坐标体系一致，不出现点位偏移。

## 2.3 设备坐标绑定需求

### 2.3.1 绑定方式

- 2D绑定：支持设备图标手动拖拽至2D地图任意位置，松手后自动获取该点位的X、Y坐标，完成设备与2D坐标的绑定，支持手动微调设备位置。

- 3D绑定：在2D视图激活某点并加载3D局部视图后，支持设备图标手动拖拽至3D场景任意位置，松手后自动获取该点位的X、Y、Z坐标，完成设备与3D坐标的绑定，支持在3D场景中调整设备的高度（Z轴）。

- 坐标拾取：系统自动完成“屏幕像素→地图坐标”的双向映射，无需用户手动输入坐标，坐标精度与矿井地图精度一致（保留2位小数）。

### 2.3.2 坐标存储与展示

- 绑定后的设备坐标（X、Y、Z）需存入系统数据库，与设备ID、矿山ID关联，支持坐标的查询、修改、删除。

- 设备绑定后，在2D/3D视图中显示设备图标及坐标信息（鼠标悬浮时展示），便于用户核对绑定位置的准确性。

- 支持批量绑定设备，可同时拖拽多个设备图标至地图对应位置，批量获取坐标并存储。

## 2.4 边缘计算适配需求

- 系统需轻量化部署，适配边缘计算设备，地图转换、坐标计算、视图渲染等核心操作均在边缘端完成，无需依赖云端，降低网络依赖。

- 支持离线运行，矿山现场无网络环境时，系统可正常加载已转换的地图文件，完成设备坐标绑定，网络恢复后自动同步数据。

- 边缘端数据存储轻量化，仅存储当前矿山的地图文件、设备信息及绑定坐标，占用存储空间≤100MB。

## 2.5 通用性与易用性需求

- 通用性：一套代码适配所有矿山，切换矿山时，仅需加载该矿山的地图文件及设备数据，无需修改系统逻辑。

- 易用性：操作流程简单，无需用户具备专业的地图知识或坐标计算能力，矿山现场人员可快速上手（拖拽、点击即可完成绑定）。

- 容错性：若设备绑定位置错误，支持一键取消绑定，重新拖拽绑定；地图加载失败时，给出明确提示（如“地图格式错误”“坐标范围异常”）。

# 3. 技术实现方案

## 3.1 核心技术栈

### 3.1.1 前端技术

- 2D地图渲染：Leaflet 或 OpenLayers（优先推荐Leaflet，轻量化、API简洁，支持GeoJSON直接渲染）。

- 3D视图渲染：Three.js 或 CesiumJS（优先推荐Three.js，轻量化适配边缘端；若需大规模3D场景，可选用CesiumJS+3D Tiles）。

- 交互逻辑：JavaScript/TypeScript，实现鼠标点击激活、设备拖拽、坐标拾取、视图切换等交互。

### 3.1.2 后端/边缘端技术

- 地图格式转换：Python（ezdxf、geopandas库）或Rust（dxf-rs库），实现DWG/DXF→GeoJSON的转换；3D格式转换（如IFC→GLB）可选用开源工具批量处理。

- 数据存储：轻量化数据库（如SQLite），存储设备信息、矿山信息、绑定坐标，适配边缘端部署。

- 坐标计算：内置坐标映射算法（无需用户手动编写），调用前端地图库API，实现屏幕像素与地图坐标的双向转换。

## 3.2 核心逻辑实现

### 3.2.1 地图加载与转换逻辑

1. 矿方提供DWG/DXF/Shapefile等带坐标的地图文件，上传至边缘端系统。

2. 系统自动执行格式转换：DWG/DXF→GeoJSON（2D），同时提取地图坐标范围（minX、maxX、minY、maxY）；若为3D地图文件（IFC/OBJ），转换为GLB/3D Tiles格式，提取Z轴范围。

3. 前端加载转换后的GeoJSON文件，渲染2D矿井总图，自动适配坐标范围，实现地图缩放、平移。

### 3.2.2 2D+3D视图切换逻辑

1. 用户在2D视图点击任意点位（如中央变电所），触发激活事件，系统识别该点位的坐标（X、Y），并加载该点位对应的3D局部模型。

2. 3D视图加载完成后，自动定位至该激活点位，保持与2D视图的坐标一致（X、Y相同），用户可在3D视图中查看空间层高，调整设备Z轴坐标。

3. 用户可通过“返回”按钮，从3D视图切换回2D总图，设备已绑定的坐标信息同步保留，点位不偏移。

### 3.2.3 设备坐标绑定逻辑

1. 2D绑定：用户进入“设备绑定模式”，拖拽设备图标至2D地图目标位置，松手后，前端调用地图API获取该点位的X、Y坐标，自动关联设备ID，存入数据库。

2. 3D绑定：在3D局部视图中，用户拖拽设备图标至目标位置，松手后，前端调用3D引擎API获取该点位的X、Y、Z坐标，覆盖/补充设备的Z轴坐标，更新至数据库。

3. 坐标映射公式（系统内置，无需手动编写）：
        

    - 2D坐标：X = minX + (mouseX / mapWidth) * (maxX - minX)；Y = minY + (mouseY / mapHeight) * (maxY - minY)（mouseX、mouseY为鼠标像素位置，mapWidth、mapHeight为2D画布宽度/高度）。

    - 3D坐标：在2D坐标基础上，增加Z轴计算，由3D引擎根据模型高程自动获取。

# 4. 交互流程说明

## 4.1 整体交互流程

1. 系统启动，边缘端加载当前矿山的2D地图（GeoJSON），渲染矿井总图。

2. 用户进入“设备管理”模块，选择需要绑定坐标的设备，进入“设备绑定模式”。

3. 2D绑定：拖拽设备图标至2D地图目标位置，松手后完成2D坐标（X、Y）绑定，系统提示绑定成功并显示坐标。

4. （可选）3D精确定位：在2D地图上点击设备绑定位置（或其他需要查看的点位），激活该点，系统自动加载对应的3D局部视图。

5. 3D绑定：在3D视图中，拖拽设备图标至精准位置，松手后完成3D坐标（X、Y、Z）绑定，系统更新设备坐标并提示成功。

6. 绑定完成后，用户可在2D/3D视图中查看设备位置及坐标，支持修改、删除绑定信息。

## 4.2 关键交互细节

- 激活事件：2D视图中，点击任意点位后，该点位高亮显示（如红色圆点），同时弹出提示“已激活该点位，正在加载3D视图”，加载完成后自动切换至3D视图。

- 设备拖拽：设备图标拖拽过程中，实时显示当前鼠标位置对应的坐标（预览），松手后确认绑定，坐标固定。

- 视图切换：3D视图中，点击“返回2D总图”按钮，自动切换回2D视图，设备点位保持不变；2D视图中，点击已激活的点位，可再次进入3D视图。

- 错误处理：若拖拽设备至地图外，系统提示“请在地图范围内绑定设备”；若3D视图加载失败，提示“该点位无对应3D模型，请更换激活点”。

# 5. 行业适配说明

## 5.1 矿山地图格式适配

矿山提供的地图格式以DWG/DXF为主（99%矿山可提供），系统优先支持该类格式的转换；对于Shapefile、MapGIS格式，采用对应开源库进行转换，确保坐标信息不丢失；对于无坐标的图片格式，系统不支持坐标绑定，需提示矿方提供带坐标的地图文件。

## 5.2 坐标系适配说明

每个矿山均有独立的矿井坐标系（自己定义原点、轴向），系统无需用户手动配置，通过解析转换后的地图文件，自动获取坐标范围和坐标系参数，实现“一矿一配置”，无需修改系统代码，确保不同矿山的通用性。

## 5.3 性能适配说明

针对矿山边缘计算设备性能有限的问题，系统采用轻量化设计：2D地图使用GeoJSON（小体积），3D仅加载局部模型（不加载全矿3D），坐标计算、地图转换均在边缘端完成，避免云端依赖，确保系统在矿山现场稳定运行。

# 6. 约束与注意事项

- 地图文件约束：矿方提供的地图文件必须带有真实坐标（无坐标文件无法实现设备绑定），且地图内部坐标统一，无偏差。

- 3D模型约束：3D局部模型需与2D地图坐标系一致，否则会出现点位偏移；建议矿方提供与2D地图对应的局部3D模型，确保坐标匹配。

- 操作约束：设备绑定需在地图范围内进行，超出地图范围的绑定操作无效；3D绑定需先激活2D点位，否则无法加载3D视图。

- 数据同步约束：边缘端数据与云端（若有）同步时，需确保坐标信息不丢失、不偏差，同步完成后需校验设备点位位置。
