# GitHub 分支保护规则

## 配置路径

GitHub 仓库 → Settings → Branches → Add branch protection rule

## main 分支保护规则

### 基本信息

```
Branch name pattern: main
```

### 保护设置

```
☑ Require a pull request before merging
  ☑ Require approvals
    Required number of approvals before merging: 1
    ☑ Dismiss stale pull request approvals when new commits are pushed
    ☑ Require review from Code Owners
    
☑ Require status checks to pass before merging
  ☑ Require branches to be up to date before merging
  Status checks that are required:
    - frontend-check
    - rust-check
    
☑ Require linear history
  (禁止 merge commit，保持历史整洁)

☑ Do not allow bypassing the above settings
  (即使是管理员也必须遵守规则)
```

### 说明

| 规则 | 目的 |
|------|------|
| Require PR | 禁止直接推送，必须通过 PR |
| Require approvals | 需要至少 1 人审核通过 |
| Status checks | CI 检查通过才能合并 |
| Linear history | 保持提交历史线性，便于回溯 |
| No bypass | 管理员也必须遵守规则 |

## develop 分支保护规则

### 基本信息

```
Branch name pattern: develop
```

### 保护设置

```
☑ Require a pull request before merging
  ☑ Require approvals
    Required number of approvals before merging: 1
    
☑ Require status checks to pass before merging
  ☑ Require branches to be up to date before merging
  Status checks that are required:
    - frontend-check
    - rust-check
```

### 说明

develop 分支保护相对宽松：
- 不要求 linear history（允许 merge commit）
- 允许管理员绕过（紧急修复时）

## Code Owners 配置

创建 `.github/CODEOWNERS` 文件：

```
# 默认所有代码由以下人员审核
* @burrs2916

# 前端代码
/src/ @burrs2916

# 后端代码
/src-tauri/ @burrs2916

# CI 配置
/.github/workflows/ @burrs2916

# 文档
/docs/ @burrs2916
/README.md @burrs2916
```

## 配置步骤

### 1. 进入仓库设置

```
https://github.com/burrs2916/biosphere-spatialdata-system/settings/branches
```

### 2. 添加 main 分支保护

1. 点击 "Add branch protection rule"
2. 输入分支名称：`main`
3. 勾选上述保护设置
4. 点击 "Create"

### 3. 添加 develop 分支保护

1. 点击 "Add branch protection rule"
2. 输入分支名称：`develop`
3. 勾选上述保护设置
4. 点击 "Create"

### 4. 创建 CODEOWNERS 文件

在仓库根目录创建 `.github/CODEOWNERS` 文件。

## 验证配置

配置完成后，尝试：

1. 直接推送到 main → 应该被拒绝
2. 创建 PR 到 main → 需要通过 CI 和审核
3. 创建 PR 到 develop → 需要通过 CI 和审核

## 注意事项

1. **管理员绕过**：建议 main 分支禁止管理员绕过
2. **审核人员**：确保有足够的审核人员
3. **CI 稳定性**：确保 CI 检查稳定可靠
4. **紧急修复**：develop 分支可允许管理员绕过，用于紧急修复
