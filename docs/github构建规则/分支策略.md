# 分支策略

本项目采用 **Git Flow 简化版** 分支策略。

## 分支结构

```
main (生产/稳定版)
  │
  ├── v0.1.0 (Tag) ──→ GitHub Release (正式版)
  ├── v0.2.0 (Tag)
  │
develop (开发/测试版)
  │
  ├── feature/cad-layer-edit ──→ 功能分支
  ├── feature/map-integration
  ├── fix/canvas-blur ──→ 修复分支
```

## 分支类型

| 分支 | 用途 | 生命周期 | 保护 |
|------|------|----------|------|
| `main` | 生产环境，稳定版本 | 永久 | ✅ 保护 |
| `develop` | 开发环境，集成测试 | 永久 | ✅ 保护 |
| `feature/*` | 功能开发 | 临时 | ❌ |
| `fix/*` | Bug 修复 | 临时 | ❌ |
| `release/*` | 发布准备 | 临时 | ❌ |

## 分支命名规范

### 功能分支

```
feature/<功能名称>

示例:
feature/cad-layer-edit
feature/map-integration
feature/export-pdf
```

### 修复分支

```
fix/<问题描述>

示例:
fix/canvas-blur
fix/memory-leak
fix/login-error
```

### 发布分支

```
release/<版本号>

示例:
release/0.2.0
release/1.0.0
```

## 分支操作规则

### 创建分支

```bash
# 功能开发
git checkout develop
git checkout -b feature/new-feature

# Bug 修复
git checkout develop
git checkout -b fix/bug-description

# 发布准备
git checkout develop
git checkout -b release/0.2.0
```

### 合并分支

```bash
# 功能分支 → develop (通过 PR)
# 修复分支 → develop (通过 PR)
# develop → main (通过 PR)
```

### 删除分支

```bash
# 功能分支合并后删除
git branch -d feature/new-feature
git push origin --delete feature/new-feature
```

## 注意事项

1. **禁止直接推送到 main**：所有变更必须通过 PR 合并
2. **禁止直接推送到 develop**：所有变更必须通过 PR 合并
3. **功能分支命名清晰**：使用英文小写，中划线分隔
4. **及时删除已合并的分支**：保持分支列表整洁
