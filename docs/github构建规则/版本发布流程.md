# 版本发布流程

## 版本号规范

遵循 [Semantic Versioning](https://semver.org/)：

```
vMAJOR.MINOR.PATCH

MAJOR - 不兼容的 API 变更
MINOR - 向后兼容的功能新增
PATCH - 向后兼容的问题修复
```

### 示例

```
v0.1.0 → v0.1.1 (修复 bug)
v0.1.1 → v0.2.0 (新功能)
v0.2.0 → v1.0.0 (重大更新/稳定版)
```

## 发布类型

### 1. 预发布 (Prerelease)

**触发条件**：Push to `main` 分支

**特点**：
- 自动构建
- 标记为 prerelease
- 版本号格式：`app-v0.1.0`

**适用场景**：
- 开发进度同步
- 内部测试
- 快速验证

### 2. 正式发布 (Release)

**触发条件**：Push tag `v*`

**特点**：
- 自动构建
- 标记为正式版
- 版本号格式：`v0.1.0`

**适用场景**：
- 功能稳定
- 准备对外发布
- 版本里程碑

## 发布步骤

### 预发布流程

```bash
# 1. 确保 develop 分支代码稳定
git checkout develop
# 运行本地测试
pnpm tauri dev

# 2. 合并到 main
git checkout main
git merge develop

# 3. 推送到 GitHub (自动触发 prerelease 构建)
git push origin main

# 4. 在 GitHub Releases 查看构建结果
# https://github.com/burrs2916/biosphere-spatialdata-system/releases
```

### 正式发布流程

```bash
# 1. 确保 main 分支代码稳定
git checkout main

# 2. 更新版本号
# 编辑 src-tauri/tauri.conf.json 中的 version
# 编辑 package.json 中的 version

# 3. 提交版本更新
git add .
git commit -m "chore: bump version to 0.2.0"

# 4. 创建 tag
git tag v0.2.0

# 5. 推送 tag (自动触发正式发布)
git push origin v0.2.0

# 6. 推送 main 分支
git push origin main
```

## 发布检查清单

### 发布前

- [ ] 所有测试通过
- [ ] 版本号已更新
- [ ] CHANGELOG 已更新
- [ ] README 已更新
- [ ] 无未解决的 PR

### 发布后

- [ ] GitHub Release 创建成功
- [ ] 构建产物上传成功
- [ ] 下载链接可用
- [ ] 安装测试通过

## 回滚操作

如果发布出现问题：

```bash
# 1. 删除远程 tag
git push origin --delete v0.2.0

# 2. 删除本地 tag
git tag -d v0.2.0

# 3. 删除 GitHub Release
# 在 GitHub Releases 页面手动删除

# 4. 修复问题后重新发布
git tag v0.2.0
git push origin v0.2.0
```
