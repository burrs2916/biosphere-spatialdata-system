# 开发工作流程

## 日常开发流程

### 1. 开始新功能开发

```bash
# 确保本地 develop 最新
git checkout develop
git pull origin develop

# 创建功能分支
git checkout -b feature/new-feature

# 开始开发
pnpm tauri dev
```

### 2. 开发过程中的提交

```bash
# 提交代码
git add .
git commit -m "feat: add new feature"

# 推送到远程
git push origin feature/new-feature
```

### 3. 创建 Pull Request

1. 访问 GitHub 仓库
2. 点击 "Compare & pull request"
3. 填写 PR 信息：
   - Title: `feat: add new feature`
   - Body: 描述功能内容和测试情况
4. 选择目标分支：`develop`
5. 点击 "Create pull request"

### 4. PR 审核和合并

```bash
# 等待 CI 检查通过
# 等待至少 1 人审核通过

# 审核通过后，合并 PR
# (在 GitHub PR 页面操作)

# 删除功能分支
git checkout develop
git pull origin develop
git branch -d feature/new-feature
```

## Bug 修复流程

### 1. 创建修复分支

```bash
git checkout develop
git pull origin develop
git checkout -b fix/bug-description
```

### 2. 修复并测试

```bash
# 修复代码
# 运行测试
pnpm tauri dev
pnpm exec tsc --noEmit
cd src-tauri && cargo test
```

### 3. 提交并创建 PR

```bash
git add .
git commit -m "fix: resolve bug description"
git push origin fix/bug-description

# 在 GitHub 创建 PR 到 develop
```

## 发布流程

### 开发版本发布

```bash
# 1. develop → main
git checkout main
git pull origin main
git merge develop
git push origin main

# 2. 等待 CI 构建完成
# 3. 在 Releases 查看构建结果
```

### 正式版本发布

```bash
# 1. 更新版本号
# 编辑 src-tauri/tauri.conf.json
# 编辑 package.json

# 2. 提交版本更新
git checkout main
git add .
git commit -m "chore: bump version to 0.2.0"
git push origin main

# 3. 创建并推送 tag
git tag v0.2.0
git push origin v0.2.0

# 4. 等待 CI 构建和发布
```

## 紧急修复流程

### 热修复 (Hotfix)

```bash
# 1. 从 main 创建热修复分支
git checkout main
git pull origin main
git checkout -b fix/hotfix-description

# 2. 修复问题
git add .
git commit -m "fix: critical bug fix"

# 3. 创建 PR 到 main
git push origin fix/hotfix-description

# 4. 合并后同步到 develop
git checkout develop
git merge main
git push origin develop
```

## 提交信息规范

遵循 [Conventional Commits](https://www.conventionalcommits.org/)：

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 类型 (type)

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: add CAD layer editing` |
| `fix` | Bug 修复 | `fix: resolve canvas blur` |
| `docs` | 文档 | `docs: update README` |
| `style` | 格式化 | `style: format code` |
| `refactor` | 重构 | `refactor: simplify logic` |
| `test` | 测试 | `test: add unit tests` |
| `ci` | CI 配置 | `ci: update workflow` |
| `chore` | 其他 | `chore: update deps` |

### 作用域 (scope)

```
feat(cad): add layer editing
fix(canvas): resolve blur issue
docs(readme): update installation guide
```

### 示例

```bash
# 简单提交
git commit -m "feat: add export PDF feature"

# 带作用域
git commit -m "feat(cad): add layer editing support"

# 带正文
git commit -m "fix: resolve memory leak" -m "The issue was caused by..."
```

## 本地测试流程

### 前端测试

```bash
# TypeScript 类型检查
pnpm exec tsc --noEmit

# 运行开发服务器
pnpm dev
```

### 后端测试

```bash
cd src-tauri

# 格式检查
cargo fmt -- --check

# Clippy 检查
cargo clippy --all-features -- -W clippy::all

# 单元测试
cargo test --all-features

# 运行应用
cargo tauri dev
```

### 集成测试

```bash
# 运行完整应用
pnpm tauri dev

# 测试主要功能:
# - CAD 文件加载
# - 场景编辑
# - 组件管理
# - 数据源连接
```

## 常见问题

### 1. CI 检查失败

```bash
# 本地运行相同的检查
pnpm exec tsc --noEmit
cd src-tauri && cargo clippy --all-features -- -W clippy::all
```

### 2. 合并冲突

```bash
# 更新 develop
git checkout develop
git pull origin develop

# 回到功能分支并合并
git checkout feature/my-feature
git merge develop

# 解决冲突后提交
git add .
git commit -m "chore: resolve merge conflicts"
```

### 3. 分支落后于远程

```bash
# 拉取并变基
git checkout feature/my-feature
git fetch origin
git rebase origin/develop
```
