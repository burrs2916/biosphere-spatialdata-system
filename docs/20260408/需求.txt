二、模块划分与职责
模块	职责	特点
Header	系统全局控制	场景切换、全屏、用户信息、搜索
Sidebar	工具面板	场景列表、组件库、绑定模式切换、告警、历史事件
Main Scene	大屏渲染	渲染空间数据、地图、3D视图、组件绑定、Layer叠加
Footer	状态栏	数据同步状态、刷新状态、告警信息、操作提示
三、前端交互设计原则
组件化
图表、表格、指标卡、地图元素、文本、按钮等都独立成组件
可拖拽、缩放、绑定 RuntimeState
Layer 层级管理
支持多场景/多 Layer（例如矿山巷道、设备拓扑、城市园区）
Layer 可单独显示或叠加渲染
数据绑定
RuntimeState / Pipeline 数据 → 组件
绑定配置存储在 SQLite → Sled 缓存 → 前端订阅
视图切换
2D ↔ 3D 或任意空间视图
保持坐标一致，Layer/组件状态同步
交互与操作
拖拽组件布局 → 保存布局
拖拽数据点 / 设备 → 更新 RuntimeState
点击事件 → 弹窗 / 高亮 / 切换 Layer
可扩展性
后续可增加轨迹展示、热力图、动画效果
可绑定不同类型空间数据（矿井、园区、城市）
四、前端目录结构建议
src/
├─ components/        # 可复用 UI 组件
│   ├─ charts/        # 图表类组件
│   ├─ maps/          # 2D/3D 地图组件
│   ├─ metrics/       # 指标卡
│   ├─ tables/        # 表格
│   └─ ui/            # 按钮、弹窗、标签
├─ pages/             # 页面级组件：DashboardPage, SceneEditorPage
├─ layouts/           # Header, Sidebar, Footer 布局
├─ store/             # 全局状态管理：RuntimeState, SceneStore
├─ services/          # Tauri IPC 服务调用
├─ hooks/             # 自定义 hook: useScene, useBinding
├─ utils/             # 工具函数：Grid / 坐标映射 / 格式转换
└─ styles/            # Tailwind / SCSS 样式
五、核心渲染机制
Scene DSL → Layer → Component
Scene 定义 Layer（地图、设备、拓扑）
Layer 渲染组件（Chart, Table, Metric, Map, Text）
组件绑定 RuntimeState，前端订阅更新
布局与拖拽
网格化布局（react-grid-layout 或自定义 Canvas/Grid）
用户拖拽组件 → 保存布局到 SQLite
交互逻辑
点击 → 激活 Layer / 弹窗 / 数据绑定
拖拽 → 更新 RuntimeState → Sled 写入 → 前端刷新

💡 总结：

左侧工具面板：可复用，支持多场景、多组件、多数据源
主显示区：Layer 化渲染空间数据，可绑定 RuntimeState，支持2D/3D视图
顶部/底部：全局控制 + 状态反馈
数据绑定与布局：完全可配置 → 存 SQLite → Sled 缓存
